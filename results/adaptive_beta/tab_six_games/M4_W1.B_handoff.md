# Phase VIII M4 W1.B handoff — 3 external baseline agents

## Summary

- Created `experiments/adaptive_beta/baselines.py` (NEW; ~600 lines)
  containing the three external baseline agents required by Phase
  VIII spec §6.3:
  - `RestartQLearningAgent` — vanilla Q-learning + full-table reset
    on (a) rolling-mean-return drop ≥ `restart_drop` from the
    all-time best (over `restart_window` completed episodes),
    (b) NaN in Q, or (c) `q_abs_max > divergence_threshold`.
  - `SlidingWindowQLearningAgent` — online Q-learning + a FIFO
    `(s, a, r, s', absorbing)` buffer of length `window_size`. At
    episode boundaries, **rows of states absent from the buffer are
    reset to `q_init`** (the simpler approximation explicitly
    permitted by the M4 brief; the alternative full-window replay
    is documented and rejected as O(window·S·A)/episode).
  - `TunedEpsilonGreedyQLearningAgent` — vanilla Q-learning with
    the tuned default `linear_epsilon_schedule(start=1.0, end=0.01,
    decay_episodes=2000)`. Constructor accepts a custom
    `epsilon_schedule` (override path) and pins the tuned constants
    as class attributes (`TUNED_EPSILON_START`, `TUNED_EPSILON_END`,
    `TUNED_EPSILON_DECAY_EPISODES`) so tests can assert on the
    canonical defaults.
- All three agents implement the runner-uniform interface
  (`select_action`, `begin_episode`, `step`, `end_episode`, `Q`
  property) that mirrors `AdaptiveBetaQAgent`'s public surface.
  Stage-2 runners (M7) can dispatch them through the same loop.
- State / action extraction uses `int(np.asarray(x).flat[0])` per
  `tasks/lessons.md` #28 (factored into a private `_as_int` helper).
- Argument validation is shared via `_validate_common_args` —
  `n_states > 0`, `n_actions > 0`, `gamma ∈ [0, 1)`,
  `learning_rate > 0`. Per-class validation (positive
  `restart_window`, `restart_drop`, `divergence_threshold`,
  `window_size`) is in each constructor.

## Design choices documented in the file

1. **Restart trigger evaluation point.** The restart predicate is
   evaluated at `begin_episode()` (i.e. between episodes), so
   `divergence_event` from the *prior* episode and `restart_event`
   for the *current* episode are reported in distinct diag rows
   (no double-counting). The all-time-best rolling mean is cleared
   on reset to prevent immediate re-triggering against a stale
   benchmark.
2. **Rolling-best gate.** The rolling-mean-return drop trigger is
   only evaluated once `len(_returns_history) >= restart_window`
   AND a "best" has been established. Partial-window "bests" are
   noisy and would never be re-attainable.
3. **Sliding-window state-eviction (vs full replay).** The brief
   explicitly allows the simpler approximation: do online TD
   updates, then at episode boundary reset rows of states absent
   from the FIFO. Rationale documented inline: full-replay is
   O(window · S · A) per episode and dominates training; this
   approximation is O(window + S) per episode and retains the
   qualitative effect (forgetting old experience). Both endpoints
   of every retained transition (`s` and `s'`) count as "visited".
4. **Tuned-ε class attributes.** The tuned constants
   (`start=1.0`, `end=0.01`, `decay_episodes=2000`) are class
   attributes so M4-W2 tests can assert on them without
   re-encoding magic numbers. The constructor accepts an explicit
   `epsilon_schedule=None` to instantiate the tuned default and
   any non-`None` callable to override.
5. **`rng` is required.** `TunedEpsilonGreedyQLearningAgent`
   raises `ValueError` if `rng=None` (no implicit seed) — same
   reproducibility-default discipline as `AdaptiveBetaQAgent`. The
   brief allows `rng=None` in the *signature* but a runtime check
   forbids `None` at construction time.

## Test invariants for `test-author` (M4 wave 2 — `test_baselines.py`)

Per spec §13.4 item 2.

### `RestartQLearningAgent`

1. **Constructor validation.** `n_states <= 0`, `n_actions <= 0`,
   `gamma < 0`, `gamma >= 1`, `learning_rate <= 0`,
   `restart_window <= 0`, `restart_drop <= 0`,
   `divergence_threshold <= 0` each raise `ValueError`.
2. **NaN trigger.** Inject NaN into Q during ep N → ep N reports
   `divergence_event=True`; ep N+1 reports `restart_event=True`
   AND `nan_count=0` (Q is reset to `q_init`).
3. **Divergence-threshold trigger.** With small
   `divergence_threshold` and a value above it inserted into Q,
   ep N+1 reports `restart_event=True`.
4. **Rolling-mean drop trigger.** Feed `restart_window` episodes
   with a high reward, then a streak of low-reward episodes such
   that the rolling mean drops by ≥ `restart_drop` from the
   established best → restart fires. Episodes 0..`restart_window-1`
   never trigger (window not yet full).
5. **Restart clears `_best_rolling_mean`.** After a triggered
   restart, `best_rolling_mean_return` in the diag is `NaN` until
   the next full window establishes a new best.
6. **Vanilla TD update is exact.** With `epsilon=0` (greedy) and
   a deterministic 2-state MDP, the post-step Q value matches the
   hand-computed `Q + α(r + γ max Q' − Q)` to within float
   tolerance.
7. **`int(state)` regression.** Pass `state=np.array([2])`,
   `action=np.array([0])`, `next_state=np.array([3])` — no
   `TypeError`. (lessons.md #28)
8. **Property `Q` is the live table.** `agent.Q is agent._Q`
   before AND after a restart (i.e. the property always reflects
   the current table).
9. **Tie-break.** With Q[s] = [1.0, 1.0, 1.0] and `epsilon=0`,
   `select_action(s, 0)` returns 0 (lowest action index;
   `np.argmax` semantics).
10. **Episode-mismatch assertion.** Calling `step(..., episode_index=2)`
    after `begin_episode(1)` raises `AssertionError`.

### `SlidingWindowQLearningAgent`

1. **Constructor validation.** Same shared checks +
   `window_size <= 0` raises `ValueError`.
2. **FIFO discipline.** After `2 * window_size` transitions, the
   buffer length equals `window_size` (deque maxlen contract).
3. **State-eviction discipline.** With `n_states=10`,
   `window_size=5`, visit only states {0, 1} for 5 transitions,
   manually corrupt rows for unvisited states → `end_episode`
   resets the corrupted rows to `q_init` and reports
   `n_states_reset=8`.
4. **Visited states preserved.** Corrupting a row for a *visited*
   state and calling `end_episode` does NOT reset that row.
5. **Vanilla TD update is exact** (same as restart agent test 6).
6. **`int(state)` regression** (same as restart agent test 7).
7. **Empty-buffer no-op.** `begin_episode(0); end_episode(0)`
   without any `step` calls reports `n_states_reset=0` (the
   no-op branch).

### `TunedEpsilonGreedyQLearningAgent`

1. **Tuned defaults.** With `epsilon_schedule=None`, ε(0)=1.0,
   ε(2000)=0.01, ε(5000)=0.01 (clamped at floor). Class
   attributes `TUNED_EPSILON_START`, `TUNED_EPSILON_END`,
   `TUNED_EPSILON_DECAY_EPISODES` equal 1.0, 0.01, 2000 exactly.
2. **Override path.** Passing a custom `epsilon_schedule` makes
   ε(0) take the custom value (regression against silently
   ignoring the override).
3. **`rng=None` is rejected.** `TunedEpsilonGreedyQLearningAgent(...,
   rng=None)` raises `ValueError`.
4. **Constructor validation** (shared checks).
5. **Vanilla TD update is exact** (same as restart agent test 6).
6. **`int(state)` regression** (same as restart agent test 7).
7. **`epsilon` field in step diag.** `step(...)["epsilon"]`
   matches `epsilon_schedule(episode_index)` exactly.

### Cross-baseline invariants (parametrized over all 3)

8. **Common surface.** Every agent has callable
   `select_action(state, episode_index)`, `begin_episode(e)`,
   `step(s, a, r, ns, absorbing, e)`, `end_episode(e)`, and a
   `Q` property returning a numpy array of shape
   `(n_states, n_actions)` with dtype `float64`.
9. **Episode-index discipline.** `step` and `end_episode` raise
   `AssertionError` when called with an `episode_index` that
   doesn't match the most-recent `begin_episode(e)`.
10. **Diagnostics dtype.** All numeric fields in the
    `end_episode` diag are JSON-serialisable Python scalars
    (no numpy scalars leaking) — important for downstream
    `metrics.npz` aggregation.
11. **β=0 / no-β.** None of these baselines emit β; their step
    diag has no `beta_used` / `td_target` deviation from the
    classical Bellman backup. (Counter-test for §6.3 — these
    are the *external* baselines.)
