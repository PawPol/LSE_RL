## Summary

Implemented the four new per-episode β schedules required by Phase VIII
Stage 5 / M4 W1.A:

- **`OracleBetaSchedule`** (`METHOD_ORACLE_BETA = "oracle_beta"`).
  Reads `episode_info["regime"]` from the env's per-episode info dict
  and looks up β via the configured `regime_to_beta` mapping. Raises
  `KeyError` on missing `episode_info` / missing `regime` key /
  unknown regime — the only schedule allowed to raise on missing data
  per spec §6.6.
- **`HandAdaptiveBetaSchedule`** (`METHOD_HAND_ADAPTIVE_BETA =
  "hand_adaptive_beta"`). Pre-registered rule
  `β_{e+1} = sign(Ā_e) · β₀ · min(1, |Ā_e| / A_scale)` with pinned
  defaults `β₀ = 1.0`, `A_scale = 0.1`, `λ_smooth = 1.0` exposed as
  class-level constants for tests. No clip layer (rule already lives
  in `[-β₀, +β₀]`).
- **`ContractionUCBBetaSchedule`** (`METHOD_CONTRACTION_UCB_BETA =
  "contraction_ucb_beta"`). 7-arm UCB on the per-episode contraction
  log-ratio `M_e = log(R_{e-1} + ε) − log(R_e + ε)` where
  `R = bellman_residual` from the agent. Per-arm Welford running
  mean/std; per-arm running mean of the *standardised* reward stream
  feeds the UCB1 score `μ̂_j_std + c · √(2 ln N / N_j)` with `c = 1.0`.
  Round-robin warm-start over the 7 arms (episodes 0..6); UCB
  selection from episode 7 onward. Ties on lowest arm index (numpy
  `argmax` convention). Residual smoothing window default = 1 (no
  smoothing); configurable.
- **`ReturnUCBBetaSchedule`** (`METHOD_RETURN_UCB_BETA =
  "return_ucb_beta"`). Same 7-arm UCB scaffolding as ContractionUCB,
  but the per-episode raw reward is `episode_return` and the UCB
  constant is the canonical UCB1 `c = √2`. Standardisation against
  the **standardised** reward (NOT the raw return) so `c = √2` is
  valid across matrix-game return ranges. Documented in the class
  docstring per spec §6.5.

ε floor `1e-8` throughout. `np.log` used directly — NO `expm1` /
`log1p` (lessons.md #27). Single TD-update path invariant preserved:
no edits to `agents.py`. The β=0 bit-identity guard at
`agents.py:338` is untouched and fires cleanly when ContractionUCB
deploys its β=0 warm-start arm in episode 3 (smoke-verified).

## Contract realised

### Protocol & base extensions

- `BetaSchedule` Protocol now exposes three optional kwargs on
  `update_after_episode`: `episode_info: Optional[Dict] = None`,
  `bellman_residual: Optional[float] = None`,
  `episode_return: Optional[float] = None`. Default `None` keeps
  every Phase VII callsite (notably
  `AdaptiveBetaQAgent._end_episode` line 239) backward-compatible.
- `_BaseSchedule.__init__` initialises three transient attributes
  (`self._last_episode_info`, `self._last_bellman_residual`,
  `self._last_episode_return`) so concrete subclasses can read them
  inside `_compute_next_beta`. Existing concrete subclasses ignore
  them.
- `_DEFAULT_HPARAMS` gained `"A_scale": 0.1` (used only by
  `HandAdaptiveBetaSchedule`; other schedules ignore it).
- New module-level constant `DEFAULT_BETA_ARM_GRID = (-2.0, -1.0,
  -0.5, 0.0, +0.5, +1.0, +2.0)` (spec §6.4).

### `OracleBetaSchedule`

| Aspect | Implementation |
|---|---|
| Constructor | `__init__(self, regime_to_beta: Dict[str, float], hyperparams=None)` |
| Episode 0 β | `0.0` (no regime observed yet) |
| Update rule | `β_{e+1} = regime_to_beta[episode_info["regime"]]` |
| Smoothing | None (oracle is exact) |
| Raises | `KeyError` on missing `episode_info` / missing `"regime"` / unknown regime |
| Allow-list keys | `{"regime_to_beta"}` (dict bypasses `_resolved_hparams`) |

| Aspect | Implementation |

### `ContractionUCBBetaSchedule`

| Aspect | Implementation |
|---|---|
| Constructor | `__init__(self, arm_grid=None, ucb_c=1.0, epsilon_floor=1e-8, residual_smoothing_window=1, hyperparams=None)` |
| Default `arm_grid` | `DEFAULT_BETA_ARM_GRID` (7 arms) |
| Per-arm reward | `M_e = log(R_{e-1} + ε) − log(R_e + ε)` (`np.log`, NO `log1p`) |
| Residual smoothing | rolling mean over last `residual_smoothing_window` raw `R` values; window=1 disables smoothing |
| Standardisation | per-arm Welford running mean/std, then per-arm running mean of the standardised stream feeds UCB |

| Warm-start | round-robin arms 0..6 in episodes 0..6 |
| UCB takes over | from episode 7 onward |
| Tie-break | lowest arm index (numpy `argmax`) |
| Allow-list keys | `{"arm_grid", "ucb_c", "epsilon_floor", "residual_smoothing_window"}` |

### `ReturnUCBBetaSchedule`

Same scaffolding as ContractionUCB plus:

| Aspect | Implementation |
|---|---|
| Constructor | `__init__(self, arm_grid=None, ucb_c=√2, epsilon_floor=1e-8, hyperparams=None)` |
| Per-arm reward | `episode_return` (sum of agent rewards over the episode), passed via `update_after_episode(... episode_return=...)` |
| UCB constant | `c = √2` (canonical UCB1) against **standardised** reward; documented in docstring |

## Factory extensions

`build_schedule(...)` gains four dispatch branches with strict
allow-list validation. Complex-typed kwargs (`regime_to_beta` dict,
`arm_grid` list) bypass `_resolved_hparams` (which calls `float(v)`)
and are passed directly to constructors. `OracleBetaSchedule`
construction via the factory requires `hyperparams["regime_to_beta"]`
to be present — the factory raises `ValueError` if the dict is missing
or empty.

## β = 0 bit-identity guard

The agent-level `β=0` collapse guard
(`AdaptiveBetaQAgent._step_update`, line 338) is untouched. The
ContractionUCB warm-start sequence deploys arm index 3 (β = 0.0) in
episode 3 of every run, cleanly exercising the classical-Bellman path.
The smoke test asserts `b_c3 == 0.0` exactly.

already passed at `agents.py:239`; the existing `divergence_event=`
call site continues to work unchanged.
