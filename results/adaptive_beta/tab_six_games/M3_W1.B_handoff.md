# Phase VIII M3 W1.B handoff — `convention_switching` adversary

## Summary

- Implemented `experiments/adaptive_beta/strategic_games/adversaries/convention_switching.py`
  per Phase VIII spec §5.7 (and §5.3 / `RR-ConventionSwitch` subcase).
  Class `ConventionSwitchingOpponent(StrategicAdversary)` plays a
  *pure* (degenerate) two-action strategy throughout each episode and
  switches between conventions (`0 = "left"`, `1 = "right"`) at
  episode boundaries.
- Two modes are selectable through the constructor:
  - `mode="periodic"`: deterministic flip every
    `switch_period_episodes` completed episodes.
  - `mode="stochastic"`: per-episode-boundary Bernoulli flip with
    probability `switch_prob`, drawn through `self._rng`.
- Constructor signature matches the spec verbatim:
  `__init__(self, n_actions: int = 2, mode: str = "periodic",
  switch_period_episodes: int = 100, switch_prob: float = 0.01,
  initial_convention: int = 0, seed: Optional[int] = None)`.
- `info()` exposes `regime ∈ {"left", "right"}` (consumed by the
  oracle β schedule per §6.6) plus `mode`, `switches_so_far`,
  `convention_index`, `episode_count`, `switch_period_episodes`,
  `switch_prob`. The mandatory `ADVERSARY_INFO_KEYS` set is preserved
  via `_build_info(...)`.
- `on_episode_end()` is the **only** state-mutating hook other than
  `reset()` — invoked by `MatrixGameEnv` (line 446 of `matrix_game.py`)
  after the terminal `step` of each episode. The hook increments the
  internal episode counter and, depending on mode, applies the flip.
  Within-episode actions and `info()` remain constant, satisfying
  Phase VIII §2 rule 2 ("β / regime updated between episodes only").
- Registered in `ADVERSARY_REGISTRY` under key `"convention_switching"`
  in `experiments/adaptive_beta/strategic_games/registry.py`.

## Disambiguation vs `ScriptedPhaseOpponent`

Per Phase VIII §5.7 (2026-04-30 conflict resolution 2):

| Property                                    | `ScriptedPhaseOpponent`           | `ConventionSwitchingOpponent` (this) |
|---------------------------------------------|-----------------------------------|--------------------------------------|
| Strategy form within a phase                | Mixed (sampled per step)          | Pure (constant action)               |
| Phase / regime exposed via `info["regime"]` | **No** (not in info contract)     | **Yes** — `"left"` / `"right"`       |
| Phase clock                                 | Deterministic, multi-phase cycle  | Deterministic period or Bernoulli    |
| Number of phases                            | Configurable (`>= 1`)             | Exactly 2 (binary convention)        |
| Intended subcases                           | RPS regression compat, MP, AC, SH | RR-ConventionSwitch                  |

The oracle β schedule (Phase VIII M4) reads `info["regime"]` for this
adversary; no other Phase VIII method may read `info["regime"]`
(spec §2 rule 6 / §6.6).

## Episode-clock semantics (periodic mode)

`on_episode_end()` increments `_episode_count` *before* checking the
flip predicate, so:

- `_episode_count = 0`, convention = `initial_convention` (e.g. `0` =
  `"left"`).
- After `on_episode_end()` is called for the first time:
  `_episode_count = 1`. With `switch_period_episodes = 5`, no flip
  (since `1 % 5 != 0`).
- After the 5th call: `_episode_count = 5`, predicate fires, flip
  toggles convention to `1` = `"right"`.
- After the 10th call: `_episode_count = 10`, flip back to
  `"left"`. Etc.

So the regime in force *during* episode `e` (0-indexed) is the
convention that was set after the `(e)`-th `on_episode_end` call —
equivalently, episodes `[0, P)` use the initial convention, episodes
`[P, 2P)` use the flipped convention, and so on. The smoke-test
trajectory below confirms this cadence directly.

## Determinism contract (recap for `test-author`)

- Within an episode `act(history, agent_action)` is a pure function of
  `self._convention`. It does NOT consume `self._rng`, so the action
  stream of a periodic-mode adversary is fully deterministic without
  needing a seed.
- `on_episode_end()` is the only `_rng`-consuming method (and only in
  stochastic mode). Two instances built with the same `seed` therefore
  produce byte-identical convention sequences.
- `reset(seed=...)` re-seeds and resets `_convention`,
  `_episode_count`, `_switches_so_far`, restoring the state to the
  byte-identical post-`__init__` snapshot.

## Invariants `test-author` should unit-test (M3 `test_convention_switching_adversary.py`)

1. **Registration / ABC compliance.** `"convention_switching" in
   ADVERSARY_REGISTRY`. `make_adversary("convention_switching")`
   returns a `ConventionSwitchingOpponent`. The mandatory key set
   `ADVERSARY_INFO_KEYS` is a subset of `adv.info().keys()` (regression
   for §5.2; piggybacks on `_validate_info` raising `KeyError` on
   omission).
2. **Constructor argument validation.** `n_actions != 2`, `mode` not
   in `{"periodic", "stochastic"}`, `switch_period_episodes < 1`,
   `switch_prob < 0`, `switch_prob > 1`, non-finite `switch_prob`,
   and `initial_convention not in {0, 1}` each raise `ValueError`.
3. **Pure within-episode play.** With any fixed convention, calling
   `adv.act(history)` repeatedly within one episode (no
   `on_episode_end()` between) returns the same integer every time;
   this is true regardless of `mode`. Concretely:
   `adv = ConventionSwitchingOpponent(initial_convention=1)` →
   `adv.act(GameHistory()) == 1` for all subsequent calls until an
   `on_episode_end()` causes a flip.
4. **Periodic flip cadence.** With `switch_period_episodes = P`,
   `initial_convention = 0`, episodes 0..P−1 report
   `info["regime"] == "left"`, episodes P..2P−1 report `"right"`,
   episodes 2P..3P−1 report `"left"`, …. The recorded regime is the
   value queried *before* `on_episode_end()` in the test loop (matches
   the smoke harness convention).
5. **Stochastic determinism.** Two adversaries built with the same
   integer `seed` and the same `mode="stochastic"` configuration
   produce byte-identical sequences of (action, regime,
   switches_so_far) over an arbitrary number of episodes.
6. **Stochastic flip statistics.** With `switch_prob = 0.5`, over
   ~1000 episode boundaries the empirical flip rate should be within
   3 standard errors of 0.5 (`σ ≈ √(0.5·0.5/1000) ≈ 0.016`); use a
   seeded run and a fixed tolerance band, NOT a chi-square test.
7. **`switch_prob = 0` → never flips.** Under `mode="stochastic",
   switch_prob=0`, `info["regime"]` equals the initial regime label
   for all episodes (regression against accidental boundary behaviour).
8. **`switch_prob = 1` → flips every boundary.** Under
   `mode="stochastic", switch_prob=1`, `info["regime"]` toggles every
   episode (alternates `left, right, left, right, ...`).
9. **Reset restores byte-identical state.** Construct adversary, run
   N episode boundaries (mixing `act` / `on_episode_end`), call
   `reset(seed=<original_seed>)`, then run another N episode
   boundaries: the post-reset trajectory equals the pre-reset
   trajectory. Important for paired-seed reproducibility (Phase VIII
   §13.7).
10. **`info["regime"]` derivable from convention alone.** Round-trip
    sanity: `info["regime"] == ("left", "right")[info["convention_index"]]`.
11. **`switches_so_far` increments correctly.** Periodic mode with
    `switch_period_episodes = 5` over 20 episodes ends with
    `info["switches_so_far"] == 4` (flips at episodes 5, 10, 15, 20).
    Stochastic determinism test: same seed → same final
    `switches_so_far`.
12. **`act` ignores `agent_action`.** Passing a non-`None`
    `agent_action` does not change the returned action (regression
    against future temptation to peek at the simultaneous-move
    counterfactual).
13. **`act` ignores `history`.** Pre-populating `GameHistory` with
    arbitrary `(agent_action, opponent_action, reward)` triples does
    not change the returned action.
14. **No `_rng` consumption inside `act` (periodic mode).** A
    deterministic invariant: for `mode="periodic"`, two adversaries
    with `seed=None` produce the same action stream — regression
    against accidentally moving the flip RNG into `act`.
15. **Default-arg construction.** `ConventionSwitchingOpponent()` with
    no kwargs is callable and produces a periodic, 100-episode-period,
    `initial_convention=0`, `seed=None` opponent. (Matches spec §5.7
    constructor signature.)
