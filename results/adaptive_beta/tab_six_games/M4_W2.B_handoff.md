## Failure Detail

`test_total_pulls_equals_total_episodes` expected 14 total pulls after 14
updates but observed 13. `test_each_arm_pulled_at_least_once_in_warm_start`
expected `(1, 1, 1, 1, 1, 1, 1)` after episodes 0..6 but observed arm 0 at
zero pulls.

The source behavior is consistent with the current implementation in
`ContractionUCBBetaSchedule._episode_reward()`: the first residual sets
`_prev_smoothed_residual` and returns `None`, so `_record_arm_reward()` is not
called for episode 0. The tests intentionally preserve the stricter spec
contract that pull accounting counts deployed episodes, not only episodes with
a computable contraction reward.
