## Summary

Implemented the Phase VIII long-CSV aggregator under
`experiments/adaptive_beta/tab_six_games/analysis/aggregate.py`
(spec §8.3). The aggregator walks per-run artifacts and emits a single
CSV keyed on `(phase, stage, game, subcase, method, seed, episode)`
that becomes the source of truth for tables and figures downstream.

### Public API

```python
from experiments.adaptive_beta.tab_six_games.analysis.aggregate import (
    aggregate_to_long_csv, AggregateRow, Phase8RunRoster,
    LONG_CSV_COLUMNS, PHASE_VIII_EXPECTED_COLUMNS,
)

summary = aggregate_to_long_csv(
    raw_root=Path("results/adaptive_beta/tab_six_games/raw"),
    out_path=Path("results/adaptive_beta/tab_six_games/processed/long.csv"),
    roster=None,
    *,
    include_phase_VII=False,
)
# summary -> {"total_runs", "total_episodes", "missing_runs", "schema_columns"}
```

### Contract checks (per spec §8.3)

1. **Walk discipline.** For each `run.json` under `raw_root`, parse
   it; locate the sibling `metrics.npz`. If `metrics.npz` is missing,
   append a `{run_dir, reason="metrics_npz_missing"}` entry to
   `missing_runs` and skip — never silently fill with zeros.
2. **Per-episode expansion.** Each `metrics.npz` array of length
   `n_episodes` is expanded into per-episode rows; metadata
   (`run_id`, `config_hash`, `phase`, `stage`, `game`, `subcase`,
   `method`, `seed`) is replicated from `run.json`. The `n_episodes`
   field on `run.json` (when present) overrides array-shape inference.
3. **No retroactive metric computation (lessons.md #19).** The
   aggregator NEVER recomputes Phase VIII delta metrics — runners
   write `metrics.npz` at fit-time via the metrics module, which is
   the single source of truth. Columns absent from a given
   `metrics.npz` render as `np.nan` (numeric) or `""` (string).
4. **Phase VII compatibility (M8 read-only narrative).** When
   `include_phase_VII=True`, the aggregator additionally walks
   `results/adaptive_beta/strategic/raw/` (resolved from the
   parent of `raw_root`) and tags those rows `phase="VII-B"`. Per
   spec §10.4 / addendum §10.5, Phase VII rows do NOT enter
   `Phase8RunRoster` — they are read-only narrative reference only.
5. **Schema-parity guard (lessons.md #16).** Each `metrics.npz` is
   validated against the §7.1 + §7.2 expected column set
   (`PHASE_VIII_EXPECTED_COLUMNS`, 40 entries). Foreign keys are
   recorded under `missing_runs` with `reason="schema_drift"` and the
   offending column list. The aggregator still emits rows for the
   recognised columns so partial drift does not block analysis.
6. **Output file.** `out_path` with `.csv.gz` suffix triggers
   transparent gzip; any other suffix writes plain CSV. Parents are
   created. `csv.writer` with `QUOTE_MINIMAL`. Streaming write — the
   header is emitted first, then rows are appended without buffering
   the full table in memory.
7. **Default base path (lessons.md #11).** `raw_root` defaults to
   `Path("results/adaptive_beta/tab_six_games/raw")`. Never
   `results/weighted_lse_dp/` or any non-Phase-VIII root. The default
   is verified at the public-API surface
   (`inspect.signature(aggregate_to_long_csv).parameters[...]`).
8. **CLI entry point.** `python -m
   experiments.adaptive_beta.tab_six_games.analysis.aggregate` with
   `--raw-root`, `--out`, `--include-phase-VII` flags. Compact
   summary on stdout (JSON), full `missing_runs` list on stderr.

### Loader normalisations

* The Python keyword `return` cannot be a valid `np.savez` keyword
  argument; existing runners persist it as `return_`. The loader
  accepts both `return` and `return_` and normalises to the canonical
  CSV column name `"return"` at row-build time.
* `run.json` field-name aliases: `task` ⇔ `game`, `algorithm` ⇔
  `method`, `suite` ⇔ `stage`. Phase VII payloads round-trip cleanly.
* Out-of-range episode indices (per-array shape shorter than the
  inferred episode count) yield `np.nan` rather than raising — known
  edge case for dev runs with partial logs.
