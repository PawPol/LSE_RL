## Potential function `Phi`

Documented in the module docstring AND in the `compute_potential`
helper docstring:

| Subcase                     | `Phi(a, b)`                          | Notes |
|-----------------------------|--------------------------------------|-------|
| PG-CoordinationPotential    | `1 if a == b else 0`                 | exact potential, `u_1 = u_2 = Phi` |
| PG-Congestion               | `5/6 if a == b else 1`               | Rosenthal potential of symmetric two-player congestion with payoff `1/(1 + count)`; exact potential |
| PG-BetterReplyInertia       | `1 if a == b else 0` (underlying)    | weakly acyclic â€” agent payoff has a path-dependent switch-cost `âˆ’lambdaÂ·1[a != prev_a]` that does NOT enter `Phi` (caveat in docstring) |
| PG-SwitchingPayoff          | `c_t Â· 1[a == b]`                    | exact potential per regime; `c_t in {1.0, 2.0}` switches every `switch_period_episodes` (default 100) at episode boundaries |

Better-reply monotonicity verified by smoke (Phi((0,0)) = 1 > Phi((1,0)) = 0
for the coordination subcase).

## `info["regime"]` convention

Per spec Â§5.6 + Â§5 info-dict contract:

- `PG-CoordinationPotential` â†’ `info["regime"] = "PG-CoordinationPotential"`
- `PG-Congestion`            â†’ `info["regime"] = "PG-Congestion"`

`asymmetric_coordination` precedent).

## Invariants `test-author` should unit-test (M2 `test_potential.py`)

1. **Registration:** `"potential" in GAME_REGISTRY` after import of
   `registry`. Update `EXPECTED_GAMES` in `test_registry.py` to include
   `"potential"` (and `"soda_uncertain"` once W1.A lands).
2. **Canonical sign:** `env.env_canonical_sign == "+"` for every
   subcase produced by `build(subcase=..., ...)`.
3. **`info["regime"]` shape:** all four subcases populate
   `info["regime"]` on both `reset()` and `step()`. For
   `PG-SwitchingPayoff`, regime is in `{"scale_lo", "scale_hi"}`;
   for the others, regime equals the subcase name verbatim.
4. **Î¦ helper correctness:**
   - `compute_potential((a, b), subcase="PG-CoordinationPotential", m=m) == 1[a==b]`.
   - `compute_potential((a, b), subcase="PG-Congestion", m=m)` returns
     `5/6` if `a == b` else `1`.
   - `compute_potential((a, b), subcase="PG-SwitchingPayoff", scale=c_t)`
     returns `c_t * 1[a==b]`.
   - Out-of-range action profile raises `ValueError`.
   - Unknown subcase raises `ValueError`.
5. **Realised-reward correctness:**
   - PG-CoordinationPotential: `r_t == Phi(a_t, b_t)` (exact equality).
   - PG-Congestion: `r_t == 1/(1 + count(a_t in {a_t, b_t}))` â€” i.e.
     `1/2` if `a != b`, `1/3` if `a == b`.
   - PG-BetterReplyInertia (`lambda > 0`):
     `r_t == Phi(a_t, b_t) âˆ’ lambda * 1[a_t != prev_agent_action]`,
     and the very first step of any episode (post-`reset`) takes no
     penalty.
   - PG-SwitchingPayoff: realised reward equals
     `c_t * Phi(a_t, b_t)` with `c_t` flipping every
     `switch_period_episodes` episodes at episode boundaries.
6. **Inertia signal in `info`:** `info["last_agent_action"]` is `None`
   immediately after `reset()` and equals the previous step's
   `agent_action` thereafter (within an episode), reset on episode
   boundary.
7. **Switch clock determinism:** `info["regime"]` for
   `PG-SwitchingPayoff` is a deterministic function of
   `episode_index // switch_period_episodes` (parity-of-block).
8. **`m` configurable:** factory accepts `m in {2, 3, 4, ...}`; envs
   built with different `m` have correctly-shaped payoff matrices
   (shape `(m, m)`) and reject adversaries with `n_actions != m`.
9. **`lambda_inertia = 0.0` reduces to coordination:** with
   `subcase="PG-BetterReplyInertia", lambda_inertia=0.0`, realised
   reward equals `Phi(a, b)` exactly (no penalty regardless of `prev_a`).
10. **Factory `kwargs` hygiene:** unknown kwargs raise `TypeError`
    (matches the precedent in other games' factories).
