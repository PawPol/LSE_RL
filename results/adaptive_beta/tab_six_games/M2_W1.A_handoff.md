## Subcase contract realised (spec §5.5)

| Subcase | Payoff (m=3 default) | `info["regime"]` |
|---|---|---|
| `SO-Coordination` | `P[i,j] = 1[i==j]` (symmetric) | constant `"coordination"` |
| `SO-AntiCoordination` | `P[i,j] = 1[i!=j]` (symmetric) | constant `"anti_coordination"` |
| `SO-ZeroSum` | agent: `+1` if `(i-j) mod m == 1`, `-1` if `== m-1`, else `0`; opp = neg | constant `"zero_sum"` |
| `SO-BiasedPreference` | `P[0,0]=2`, `P[i,i]=1` for `i>0`, else 0 (symmetric) | constant `"biased_preference"` |
| `SO-TypeSwitch` | rotates active matrices each episode | rotates among the 4 strings |

Rotation rule for `SO-TypeSwitch` (deterministic, no env RNG draw):

    regime[e] = ["coordination", "anti_coordination",
                 "zero_sum", "biased_preference"][(e // P) % 4]

with `P = switch_period_episodes` (default 100), `e = self._episode_index`.
The clock is fully derivable from `_episode_index` alone — no hidden state.

## Architecture

- `SodaUncertainEnv` is a thin subclass of `MatrixGameEnv`. Its only
  responsibilities are (a) inject `info["regime"]` into both `reset` and
  `step` info dicts, and (b) for `SO-TypeSwitch`, swap
  `self._payoff_agent` / `self._payoff_opponent` at the start of each
  episode according to the per-episode clock. All other behaviour
  (history, adversary lifecycle, state encoder, MDPInfo, gamma) is
  inherited unchanged.
- `env.env_canonical_sign = None` — wrong-sign / adaptive-magnitude-only
  schedules will raise from the schedule factory against this env (spec
  §6.1, §22.3 precedent), as required.
- The default state encoder (`make_default_state_encoder(horizon, m)`)
  exposes only `(timestep, prev_opponent_action)` — the regime is NOT
  observable through the state. Per spec §6.6 only `OracleBetaSchedule`
  may read `info["regime"]`; non-oracle schedules see no leakage.

### Deeper sanity checks (auxiliary, all PASS)

- **Rotation period correctness.** With `switch_period_episodes=3` over
  13 episodes the realised regime sequence is
  `[coordination]*3, [anti_coordination]*3, [zero_sum]*3,
  [biased_preference]*3, [coordination]*1` (matches spec).
- **Payoff correctness, every subcase:**
  - `SO-Coordination`, agent always plays 0 over 20 steps:
    reward `= 1.0` iff opponent plays 0, else `0.0`.
  - `SO-BiasedPreference (0,0) = 2.0`; `(1,1) = 1.0`.
  - `SO-ZeroSum (0,1) = -1.0` (since `(0-1) mod 3 == 2 == m-1`);
    `SO-ZeroSum (0,2) = +1.0` (since `(0-2) mod 3 == 1`).
  - `SO-AntiCoordination (0,1) = +1.0`, `(1,1) = 0.0`.
- **`info["regime"]` present in BOTH `reset` and `step` info dicts**
  (oracle reads on every episode boundary).
- **Full-trajectory determinism:** two `SO-TypeSwitch` runs with identical
  `(seed, adversary_seed, switch_period_episodes)` produce byte-identical
  `(state, reward, regime, opponent_action)` sequences across 4 episodes.
- **Reset state shape/dtype:** `shape=(1,) dtype=int64` (matches the
  default `make_default_state_encoder` contract).
- **Reset info keys** (sorted): `['adversary_info', 'agent_action',
  'catastrophe', 'episode_index', 'game_name', 'is_shift_step',
  'opponent_action', 'phase', 'regime', 'terminal_success']` — `regime`
  present, all spec §5 mandatory keys present.
