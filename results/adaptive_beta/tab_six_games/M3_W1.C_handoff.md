## Contract realised (spec ┬¦5.7 + ┬¦10.5 + ┬¦6.6)

### Class signature (verbatim from dispatch + spec)

```python
SignSwitchingRegimeOpponent(
    n_actions: int,
    mode: str = "exogenous",                 # "exogenous" | "endogenous"
    dwell_episodes: int = 250,               # spec grid {100, 250, 500, 1000}
    trigger_window: int = 50,
    trigger_threshold_high: float = 0.5,
    trigger_threshold_low: float = -0.5,
    initial_regime: str = "plus",            # "plus" | "minus"
    policy_plus_actions: Optional[Callable] = None,   # uniform fallback
    policy_minus_actions: Optional[Callable] = None,  # uniform fallback
    seed: Optional[int] = None,
)
```

### Switching modes

| Mode | Trigger | Fires in `on_episode_end()` |
|---|---|---|
| `"exogenous"` | `episodes_in_current_regime >= dwell_episodes` | yes |
| `"endogenous"` | rolling agent-reward mean over the last `trigger_window` rewards crosses `trigger_threshold_high` (plus ŌåÆ minus) or `trigger_threshold_low` (minus ŌåÆ plus) | yes; window must be full to fire |

The endogenous trigger consumes ONLY the rewards delivered by the env
through `observe(...)`, never the contents of the `GameHistory` argument
to `act`. This mirrors what the M9 composite env will actually emit
(one `observe` per realised step) and avoids double-counting if the
agent passes a freshly-built history to each `act` call.

### `info()` keys

`info()` returns the full ┬¦5.2 mandatory key set

`temperature`, `model_rejected`, `search_phase`, `hypothesis_id`,
`policy_entropy`) plus the M3 W1.C narrative metadata:

- `switches_so_far : int`

- `rolling_reward : float` (mean of the trigger window; `0.0` on an
  empty window)
- `dwell_episodes`, `trigger_window`, `trigger_threshold_high`,
  `trigger_threshold_low` (the controller configuration, surfaced for
  log/plot post-hocs).

### Auxiliary sanity checks (all PASS)

- **Mandatory `info()` keys**: `ADVERSARY_INFO_KEYS.issubset(info().keys())`
  is `True`; `_validate_info` accepts the dict.
- **Reset re-seeding**: an instance constructed with `seed=42`, then
  `reset(seed=None)`, produces a byte-identical 20-step action stream
  to a freshly-constructed `seed=42` instance.
- **Endogenous flap-back**: `trigger_window=3`, `high=+0.5`, `low=-0.5`,
  initial `"plus"`, feed five `+1.0` rewards ŌåÆ flips to `"minus"`,
  `switches_so_far=1`. Then feed five `-1.0` rewards ŌåÆ flips back to
  `"plus"`, `switches_so_far=2`. (Bidirectional trigger correctness.)
- **Per-regime callable dispatch**: with `dwell_episodes=2`,
  `policy_plus_actions=lambda h: 0`, `policy_minus_actions=lambda h: 1`,
  over 8 episodes the realised opponent-action stream is
  `[0, 0, 1, 1, 0, 0, 1, 1]` ŌĆö exactly the regime clock applied to the
  per-regime policy. Each policy callable was invoked 4 times.
- **Validation errors**: `mode="nonsense"`, `dwell_episodes=0`,
  `initial_regime="bogus"`, and `trigger_threshold_high < trigger_threshold_low`
  all raise `ValueError` loudly (no silent defaults).
